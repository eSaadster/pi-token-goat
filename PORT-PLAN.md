## 1. Goal & scope

**Primary goal:** a full TS/Node port re-proven against a rewritten test suite, so the same token-saving behaviors hold in TypeScript.

**Scope recommendation (the purpose-fork):** the fork's reason for existing (`eSaadster/pi-token-goat`, active `.pi/extensions/token-goat.ts`) is a **zero-Python runtime for pi/opencode/openclaw**. ## 3. Architecture (key bets)

1. **Sync-by-default, four async seams.** The Python codebase is overwhelmingly blocking (sync SQLite, sync fs, `spawnSync`). `better-sqlite3`'s sync API was chosen *because* it matches this — zero refactor of ~30 DB/session/cache modules. Async appears only where Node forces it: (a) `child_process` streams (bash_runner/worker), (b) web-tree-sitter WASM load, (c) `@huggingface/transformers` inference, (d) the hook-dispatch watchdog (a `worker_thread` per dispatch, mandatory because Node can't interrupt a hung handler on the main thread). The viral-async anti-pattern is explicitly rejected.
2. **One types file as dependency root** — `src/types.ts` (export-type-only, zero runtime deps). Consolidates the TypedDicts/dataclasses Python scatters across 5 modules. Every src file + every bridge imports from it.
3. **Explicit reset registry** (`src/reset.ts`) as the linchpin test seam. JS has no monkeypatch, so the ~9 module-level caches Python's `conftest.py` clears MUST become an explicit registry (`clearModuleCaches()`), called in every test's `beforeEach`. Getting this wrong = flaky tests that pass alone and fail in suite.
4. **ESM entry is the purpose-fork payoff** — `dispatchHook` exported from `src/index.ts`; the pi bridge imports it in-process. CLI and ESM entry share `src/hooks-cli.ts`, so both paths exercise the same tests.
5. **Soft zero-native-deps** — better-sqlite3 + sharp accepted (prebuilt); `sqlite-vec` loads via `loadExtension` with try/catch → `embeddings_disabled` on the ~5 unsupported platforms (feature degrades, core never breaks).
6. **Deferrable Layer-7 surface** — the 17 flat/regex language adapters + stub grammars give a working index for most files, so the 9 tree-sitter grammar taxonomy ports and the embeddings integration (the two highest-risk surfaces) can ship disabled without blocking the core.

Pure functions (entropy, diff, hash, truncate) must be byte-exact; the bash filters use the ~74 bucketed test files as golden outputs; embeddings verified within float epsilon against the same ONNX checkpoint. ## 8. How to run the next layer

The execution engine is `token-goat-ts-port-layer` (a saved workflow). Invoke it per layer; it ports each module test-first and adversarially verifies parity, returning file contents. The main loop writes files, runs `tsc` + `vitest` + the parity harness, and reports. Mega-modules (bash_compress, cli) are split into framework + category sub-tasks at invocation time.

Recommended cadence: one layer per turn, review the parity report, then proceed.
